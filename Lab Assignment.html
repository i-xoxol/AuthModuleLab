The CIA Triad and Secure Software Development

The CIA triad is a fundamental model in information security that helps in the design and evaluation of security policies. It consists of three core principles:

  • Confidentiality: Ensuring that sensitive information is not disclosed to unauthorized individuals. In this lab, confidentiality is achieved by hashing passwords and protecting user data.
  • Integrity: Maintaining the consistency, accuracy, and trustworthiness of data over its entire lifecycle. In this lab, integrity is maintained by ensuring that only authorized users can modify their data (which is not implemented, but you can think about how you would do it) and by using cryptographic hashes to protect passwords from modification.
  • Availability: Ensuring that information is available when it is needed. In this lab, availability is ensured by having a functional application that allows users to register, log in, and access their profiles.

Lab Architecture

This lab is composed of several components that work together to provide a secure authentication system. Understanding the role of each component is crucial for understanding the overall security of the application.

Back-End Components (Python)

  • `api.py`: This is the main entry point of our web application. It uses the Flask framework to create a web server and define the API endpoints for handling login, registration, logout, and profile access. It also manages user sessions.
  • `auth.py`: This module contains the core authentication logic. It provides functions for hashing passwords, comparing passwords, and managing user data in the `usersdb.json` file. This separation of concerns is important for maintainability and security.
  • `user.py`: This module defines the `User` class, which represents a user in our system. It is a simple data structure that holds the username, hashed password, and salt.
  • `user_registration.py`: This module handles the user registration process. It uses functions from `auth.py` to create new users and save them to the database.
  • `usersdb.json`: This is a simple JSON file that acts as our user database. In a real-world application, you would use a more robust database system like PostgreSQL or MySQL.

Front-End Components (HTML/CSS/JavaScript)

  • `index.html`, `login.html`, `register.html`: These are the HTML files that define the structure of our web pages.
  • `style.css`: This file contains the CSS code for styling the web pages, making them visually appealing and user-friendly.
  • `script.js`, `register.js`: These JavaScript files contain the logic for handling user interactions on the front-end, such as form submissions and communication with the back-end API.

Testing Components (Python)

  • `test_auth.py`, `test_user_registration.py`, `test_api.py`: These files contain the unit tests for our application. Unit tests are crucial for ensuring the correctness and security of our code. They help us to automatically verify that our functions and endpoints behave as expected and to catch any regressions or vulnerabilities that might be introduced during development.

Your Task

Your task in this lab is to explore the provided code, run the application and the tests, and understand how the different components work together to create a secure authentication system. Follow these steps:

  1. Review the code: Start by reviewing all the files in the project. Pay close attention to the comments in the code, as they explain the purpose of each function and component.
  2. Run the application:
    • Open your terminal and navigate to the project directory.
    • Run the API using the command: `python api.py`
    • Open your web browser and go to: `http://127.0.0.1:5000`
    • Experiment with the application by registering a new user, logging in, accessing the profile page, and logging out.
  3. Run the unit tests:
    • Open your terminal and navigate to the project directory.
    • Run the unit tests using the command: `python -m unittest discover`
    • Review the test files to understand what is being tested and how the tests are structured. Notice how the tests cover different scenarios, including valid and invalid inputs, to ensure the robustness of the application.
  4. Answer the following questions:
    1. How does the application use hashing to protect user passwords? (Confidentiality)
    2. How do the unit tests help to ensure the integrity of the authentication module?
    3. How does the session management system contribute to the security of the application?
    4. What are the potential security risks of using an in-memory session store in a production environment? (Availability)
    5. How would you improve the security of this application?
